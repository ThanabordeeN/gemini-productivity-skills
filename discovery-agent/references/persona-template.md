# User Persona Template

## Format มาตรฐาน

```
──────────────────────────────────────────
PERSONA: [ชื่อสมมติ] — [Role/ตำแหน่ง]
──────────────────────────────────────────
📌 Quote: "สิ่งที่เขาพูดถึงปัญหาของตัวเอง"

Demographics:
- อายุ: [range]
- ระดับเทค: [Beginner / Intermediate / Expert]
- ใช้งานผ่าน: [Mobile / Desktop / Both]
- บริบท: [ทำงานที่ไหน ใช้เมื่อไหร่]

Goals (สิ่งที่ต้องการ):
- [goal 1]
- [goal 2]

Pain Points (ปัญหาที่เจอ):
- [pain 1]
- [pain 2]

Behavior (พฤติกรรม):
- ตอนนี้แก้ปัญหาด้วย: [วิธีเดิม]
- ความถี่ในการใช้งาน: [ทุกวัน / weekly / รายเดือน]

ความต้องการจากซอฟต์แวร์:
- Must have: [feature ที่ขาดไม่ได้]
- Nice to have: [feature ที่ดีถ้ามี]
──────────────────────────────────────────
```

---

## ตัวอย่าง Persona สำหรับระบบ HR Software

```
──────────────────────────────────────────
PERSONA: คุณนภา — HR Manager
──────────────────────────────────────────
📌 Quote: "ทุกวันต้องเปิด Excel หลายไฟล์ บางทีข้อมูลไม่ตรงกัน ทำให้เสียเวลามาก"

Demographics:
- อายุ: 30-45 ปี
- ระดับเทค: Intermediate (ใช้ Excel เก่ง แต่ไม่ชอบระบบซับซ้อน)
- ใช้งานผ่าน: Desktop เป็นหลัก, Mobile บ้างเมื่อรีบ
- บริบท: ทำงาน Office วันจันทร์-ศุกร์, ต้องรายงาน MD ทุกสัปดาห์

Goals:
- ดูข้อมูลพนักงานได้ครบในที่เดียว
- สร้าง Report ได้เร็ว ไม่ต้องรวบรวมข้อมูลเอง
- ลดขั้นตอน Approve ลาที่ยุ่งยาก

Pain Points:
- ข้อมูลพนักงานกระจายอยู่หลายไฟล์ Excel
- การ Approve ลาต้องส่ง Email ไปมา ไม่รู้ว่าอยู่ในขั้นตอนไหน
- รายงานสรุปต้องทำเองทุกเดือน ใช้เวลา 2-3 วัน

Behavior:
- ตอนนี้แก้ปัญหาด้วย: Excel + Email + Line
- ความถี่: ใช้ระบบ HR ทุกวัน

ความต้องการ:
- Must have: จัดการข้อมูลพนักงาน, ระบบลา, รายงาน
- Nice to have: Mobile app, Integration กับระบบเงินเดือน
──────────────────────────────────────────
```

## จำนวน Persona ที่แนะนำ

| ขนาดโปรเจกต์ | จำนวน Persona |
|-------------|--------------|
| MVP / Startup | 2 Personas |
| Mid-size Product | 3-4 Personas |
| Enterprise Software | 4-6 Personas |

> กฎ: ถ้ามี Persona มากกว่า 1 ตัว ให้ระบุ **Primary Persona** (คนที่สำคัญที่สุด) เสมอ
