# Monthly Report Expert

This skill provides procedural guidance for crafting high-impact monthly reports that serve as decision-making tools rather than mere data summaries.

## Core Methodologies

### 1. The Pyramid Principle (Top-Down Communication)
- **Answer First:** Lead with the executive summary and key recommendations.
- **Support:** Group supporting arguments beneath the main conclusion.
- **Evidence:** Provide data, charts, and analysis to back up arguments.

### 2. MECE (Mutually Exclusive, Collectively Exhaustive)
- Ensure problem categories and data groupings do not overlap (MECE).
- Cover the entire problem space without gaps.
- *Example:* Segment issues into **Technical**, **Process**, and **People**.

### 3. The "So What?" Test
- Every data point must have a business implication.
- *Good:* "Automation reduced processing time by 50%."
- *Better (McKinsey Style):* "Automation reduced processing time by 50%, enabling a 15% reduction in warehouse OT costs."

### 4. Automation & Process Improvement KPIs
When reporting on automation (e.g., n8n, Python, Excel-to-Web), prioritize:
- **Efficiency Gains:** Time saved (Minutes/Hours) per transaction or shipment.
- **Accuracy Rate:** Reduction in Human Error (e.g., mis-scanned items).
- **System Reliability:** Uptime and success rate of automated workflows.

## Workflow: Creating a Report

1. **Define the Objective:** What is the primary "Win" or "Challenge" this month?
2. **Synthesize Data:** Apply the "So What?" test to your raw KPIs.
3. **Structure with Pyramid Principle:**
   - Draft the Executive Summary first.
   - Organize supporting data into MECE categories.
4. **Select a Template:** Use [report-templates.md](references/report-templates.md) to choose the appropriate style for your audience.
5. **Add Actionable Insights:** Define "Next Steps" with clear ownership (Who, What, When).
